Knowledge Tree

Map What You Know

A conventional exam score compresses performance into a single percentage. It records what was secured, but tells you very little about how understanding is actually organised. A result like 74% may say what was gained, but it does not show where knowledge is secure, where it is partial, or how far a learner can travel into deeper application.

This prototype treats assessment as a knowledge tree. Multiple roots allow entry through different foundational ideas. These converge into a shared trunk of core definitions and relationships. From there, branches open into deeper applications, while the canopy holds synthesis questions with the highest reward. The aim is not just to total marks, but to make the structure of understanding visible.

The model is designed around three dimensions. Breadth is shown by traversing multiple roots and lower branches. Depth is shown by climbing into more demanding branches and canopy questions. Recovery is built in: a wrong answer need not end the route, but can redirect the learner through a shorter remedial path testing the same idea in a more accessible form.

This is a work in progress rather than a finished assessment system. The present version explores the core metaphor and mechanics: weighted routes, visible bifurcation points, and partial recovery after error. The next stage is to make the underlying design more rigorous: clearer traversal rules, clearer trade-offs between breadth and depth, and a richer end profile than a single score alone.

Prototype mechanics:
ROOT = 1 pt  ·  TRUNK = 2 pts  ·  BRANCH = 3 pts  ·  CANOPY = 4 pts
Wrong answer → remedial detour → revisit at 75%
Bifurcation points → route choice with visible trade-offs
Current build explores one possible tree, not yet a final scoring model
