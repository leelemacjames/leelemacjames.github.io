Linearisation

Making curved relationships confess their secrets
IB Physics SL/HL — Measurement, Data Processing & Experimental Technique
§1 Why Linearise?
Why not just draw the curve and be done with it?▾

Because a straight line is the only graph you can fully characterise by eye. You can judge whether data fits it, read off two numbers (gradient and intercept) that encode the physics, and estimate uncertainties by drawing max/min lines.

A curve tells you "something is happening". A straight line tells you exactly what.

Linearisation is not a mathematical trick — it is an experimental strategy. You reshape the data so that the physics falls out of the gradient and intercept of \(y = mx + c\).
What is the general strategy for linearising a relationship?▾

Start with the theoretical equation. Rearrange it into the form:

\(\underbrace{(\text{plot on } y\text{-axis})}_{y} = \underbrace{(\text{physics constant})}_{m} \times \underbrace{(\text{plot on } x\text{-axis})}_{x} + \underbrace{(\text{maybe zero})}_{c}\)

The variables you measure appear as \(y\) and \(x\). The constants you want to find appear as \(m\) (gradient) or \(c\) (intercept).

The key decision is what to plot on each axis — square, reciprocal, or logarithm — whatever turns the equation into \(y = mx + c\).

§2 Power Laws: \(y = kx^n\)
How does a log-log plot linearise a power law — and what do gradient and intercept mean?▾

Start with \(y = kx^n\). Take the natural log:

\(\ln y = n\ln x + \ln k\)

This is \(Y = mX + c\) with \(Y = \ln y\), \(X = \ln x\), \(m = n\), \(c = \ln k\).

The gradient gives you the exponent \(n\) directly. Gradient of 2 → \(y \propto x^2\). Gradient of \(-1\) → \(y \propto 1/x\). Gradient of ½ → \(y \propto \sqrt{x}\).

The \(y\)-intercept gives \(\ln k\), so \(k = e^{\text{intercept}}\).

Every power law becomes a straight line on a log-log plot. The exponent — hidden in the curvature — appears directly as the gradient. This is why log-log paper exists.
How do you linearise a pendulum: \(T = 2\pi\sqrt{\ell/g}\)?▾

Approach 1 — Square it (known power):

\(T^2 = \dfrac{4\pi^2}{g}\,\ell\)

Plot \(T^2\) vs \(\ell\). Gradient \(= 4\pi^2/g\). Intercept \(= 0\).

Approach 2 — Log both sides (testing the power):

\(\ln T = \tfrac{1}{2}\ln\ell + \ln\!\left(\dfrac{2\pi}{\sqrt{g}}\right)\)

Plot \(\ln T\) vs \(\ln\ell\). Gradient \(= \tfrac{1}{2}\). Intercept gives \(g\).

Approach 1 is cleaner when you know the power. Approach 2 is better when you're testing what the power is.
Intensity follows \(I = P/(4\pi d^2)\). How would you verify the inverse-square law?▾

Direct: Plot \(I\) vs \(1/d^2\). Straight line through origin, gradient \(= P/(4\pi)\).

Log-log: \(\ln I = -2\ln d + \ln(P/4\pi)\). Plot \(\ln I\) vs \(\ln d\). Gradient should be \(-2\).

Log-log tests the exponent rather than assuming it. If you get \(-1.87\) instead of \(-2\), you've discovered something — absorption, reflection, or a different law.
Boyle's Law: \(pV = \text{const}\). Linearisation options?▾

Option 1: Plot \(p\) vs \(1/V\). Gradient \(= nRT\). Intercept \(= 0\).

Option 2: Plot \(pV\) vs \(V\). Horizontal line → Boyle's law holds. Deviations are amplified.

Option 3: \(\ln p = -\ln V + \ln k\). Plot \(\ln p\) vs \(\ln V\). Gradient \(= -1\).

Option 2 is under-used but elegant — a curve on \(p\) vs \(1/V\) might look straight; the same data on \(pV\) vs \(V\) shows the wobble clearly.
§3 Exponential Relationships: \(y = Ae^{kx}\)
How does an ln-linear plot linearise an exponential — and what do gradient and intercept mean?▾

Start with \(N = N_0 e^{-\lambda t}\). Take the natural log:

\(\ln N = -\lambda t + \ln N_0\)
RoleExpressionWhat it tells you
\(y\)\(\ln N\)Plot on vertical axis
\(x\)\(t\)Plot on horizontal axis
\(m\)\(-\lambda\)Gradient → decay constant
\(c\)\(\ln N_0\)Intercept → initial count (\(N_0 = e^c\))

From \(\lambda\): \(t_{1/2} = \ln 2 / \lambda\).

This is the single most common IB linearisation. It appears in radioactive decay, capacitor discharge, Beer-Lambert attenuation, and cooling curves.
Capacitor discharge: \(V = V_0 e^{-t/RC}\). How to extract \(RC\)?▾
\(\ln V = \ln V_0 - \dfrac{1}{RC}\,t\)

Plot \(\ln V\) vs \(t\). Gradient \(= -1/RC\), so \(RC = -1/\text{gradient}\).

Example: gradient \(= -0.25\) s⁻¹ → \(RC = 4.0\) s. If \(R = 10\) kΩ → \(C = 400\) μF.

Using \(\log_{10}\) instead of \(\ln\) makes the gradient off by a factor of 2.303. Stick to natural logs.
\(y = A(1 - e^{-t/\tau})\) — exponential approach to a limit?▾

Rearrange first: \(A - y = Ae^{-t/\tau}\), then:

\(\ln(A - y) = \ln A - t/\tau\)

Plot \(\ln(A - y)\) vs \(t\). Gradient \(= -1/\tau\).

Isolate the exponential, then take the log. Don't try to take the log of a sum.
§4 The Two Master Plots
When do I use log-log, and when do I use ln-linear?▾
RelationshipPlot typeWhat gradient gives
\(y = kx^n\) (power law)\(\ln y\) vs \(\ln x\)The exponent \(n\)
\(y = Ae^{kx}\) (exponential)\(\ln y\) vs \(x\)The rate constant \(k\)

How to tell them apart: Curve through origin → power law (log-log). Non-zero intercept and bends → exponential (ln-linear). In doubt? Try both — only one gives a straight line.

Log-log says "related by a power" (geometry). Ln-linear says "one variable is in the exponent" (process). Fundamentally different physics.
§5 Extracting Physics from the Graph
How do you correctly read a gradient and intercept — with units?▾

Gradient units: \([\text{y-axis}] / [\text{x-axis}]\). Intercept units: \([\text{y-axis}]\).

Example: \(T^2\) (s²) vs \(\ell\) (m). Gradient = s²/m. Gradient \(= 4\pi^2/g\), so:

\(g = 4\pi^2/\text{gradient} \quad [\text{m s}^{-2}]\)

On a log graph: axes are dimensionless, gradient is a pure number (the exponent or rate constant).

IB examiners penalise gradients from data points rather than from the line. Use two points on the line, far apart.
How do you handle uncertainties on a linearised graph?▾

1. Plot data with error bars. (For \(\ln y\): error bar \(\approx \Delta y / y\).)

2. Draw best-fit, steepest, and shallowest lines through the error bars.

3. Uncertainty in gradient:

\(\Delta m = (m_{\max} - m_{\min})/2\)

4. Propagate: if gradient \(= -\lambda\), then \(\Delta\lambda = |\Delta m|\) and \(\Delta t_{1/2} = t_{1/2} \times \Delta\lambda/\lambda\).

Error bars on \(\ln y\) are asymmetric in raw units but symmetric in log units — multiplicative uncertainties become additive after taking logs.

§7 Common Pitfalls
What are the mistakes that lose marks?▾

1. Forgetting to transform units. \(T^2\) axis is s², not s. \(\ln V\) axis is dimensionless.

2. Using data points instead of the line. Gradient from the best-fit line, not from raw data.

3. Confusing \(\ln\) and \(\log_{10}\). For exponentials use \(\ln\). Using \(\log_{10}\) gives gradient off by 2.303.

4. Not checking the intercept. Zero-intercept theory + non-zero intercept data = systematic error.

5. Wrong plot type. \(\ln y\) vs \(\ln x\) tests a power law. \(\ln y\) vs \(x\) tests an exponential. Different hypotheses.

The most devastating mistake is plotting the wrong variables. Write the equation, identify \(y = mx + c\), label axes. Then plot.
Σ The Method

Step 1. Write down the theoretical equation.

Step 2. Identify: power law (\(y = kx^n\)) or exponential (\(y = Ae^{kx}\))?

Step 3. Transform: square, reciprocal, or \(\ln\) — whatever produces \(y = mx + c\).

Step 4. Calculate transformed data in a table.

Step 5. Plot with error bars. Draw best-fit, max, and min lines.

Step 6. Read gradient and intercept from the line.

Step 7. Extract the physics. State result with uncertainty and units.

Linearisation is the physicist's confession: we only trust straight lines. Everything else — exponentials, power laws, oscillations — must be rewritten until a straight line appears. The line is the interface between theory and measurement.
