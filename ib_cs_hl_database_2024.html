1(b)
Identify three reasons why it is important to enforce referential integrity.
[3]
✓ Model Answer — 1 mark per point
  • Improves data accuracy / reduces record anomalies / improves data retrieval accuracy
  • Improves data consistency — ensures records in a linked table relate to data in another table (via primary/foreign key), allowing extraction from different tables
  • Ensures a deleted record does not leave "hanging" records unlinked to other tables
  • Simplifies update/maintenance through cascade delete/update
e.g. If a police officer who interviewed a witness is deleted, they cannot be identified from the interview record — referential integrity prevents this orphan.
1(c)
Explain how a query can provide a view of the database.
[3]
✓ Model Answer — choose one cluster

Cluster 1 — how a view is defined:

  • A name is assigned to the view: CREATE VIEW view_name AS
  • Fields (columns) are selected: SELECT field1, field2 FROM table(s)
  • A condition/filter is specified: WHERE condition
  • This reduces the dataset to include only the required data

Cluster 2 — properties of a view:

  • A query/filter provides a subset of the database
  • It can be saved and reused regularly (e.g. weekly updates)
  • The view is not part of the physical schema (virtual table)
  • The view can be organised/sorted to meet reporting needs
Note: marks awarded from one cluster only — do not mix clusters.
Question 2 Secondary keys · Composite PKs · 3NF normalisation
2(a)
Define the term secondary key.
[1]
✓ Model Answer
  • A field (or combination of fields) that is used to index a table other than the primary key
  • It allows efficient searching/retrieval on non-primary-key fields
  • It does not have to be unique (unlike a primary key)
  • Also called an alternate key or index key in some contexts
2(b)(i)
INTERVIEW(WitnessID, CrimeID, OfficerID, Statement, …) — composite PK = {WitnessID, CrimeID, OfficerID} Outline why {WitnessID, CrimeID, OfficerID} is not a suitable composite primary key.
[2]
✓ Model Answer
  • A witness may be interviewed more than once by the same officer about the same crime
  • This means the combination {WitnessID, CrimeID, OfficerID} would not be unique across all rows
  • A primary key must uniquely identify each record — this composite key fails that requirement
  • Solution: add a surrogate key such as InterviewID (auto-increment) as the PK, or include DateTimeStatement in the composite
2(b)(ii)
Outline why the INTERVIEW table is not in 3NF.
INTERVIEW(WitnessID, CrimeID, OfficerID, Statement, CrimeLocation, DateTimeStatement, PoliceStation, StationPhone)
[4]
✓ Model Answer — 2NF and 3NF violations

2NF violation (partial dependency):

  • CrimeLocation depends only on CrimeID, not on the full composite key → partial dependency
  • This violates 2NF: every non-key attribute must depend on the whole key

3NF violation (transitive dependency):

  • StationPhone depends on PoliceStation, not directly on the primary key
  • This is a transitive dependency: PK → PoliceStation → StationPhone
  • 3NF requires every non-key attribute to depend only on the primary key, not on another non-key field
To reach 3NF: remove CrimeLocation to CRIME table; create a POLICESTATION(PoliceStation, StationPhone) table and reference it by FK from INTERVIEW.
2(c)
Outline why an alphanumeric data type is used for the VictimPhone field.
[2]
✓ Model Answer
  • Phone numbers may include leading zeros (e.g. 01029…) — storing as integer would strip them
  • Phone numbers may include non-numeric characters: spaces, hyphens, +44 country codes, brackets
  • No arithmetic is performed on phone numbers, so numeric type offers no benefit
  • Alphanumeric preserves the number exactly as entered

Question 3 Concurrency · Centralised DB · Data mining · Matching · GDPR
3(a)
Two police officers attempt to update a witness statement at the same time. Describe how data locking can be used to deal with this situation.
[2]
✓ Model Answer
  • When Officer A begins editing the record, the DBMS places a lock on that record
  • Officer B's attempt to edit the same record is blocked / queued until the lock is released
  • Once Officer A commits/saves, the lock is released and Officer B can then access and update the record
  • This prevents a lost update anomaly where one officer's changes overwrite the other's
Row-level locking is preferred over table-level locking as it allows other witnesses' records to remain accessible.
3(b)
Discuss the advantages and disadvantages of using a centralised crime database.
[5]
✓ Model Answer — balanced discussion required

Advantages:

  • Single source of truth — all forces access the same, up-to-date data; no inconsistency
  • Easier to search nationally — officers can identify suspects/witnesses across regions
  • Simpler to maintain, update, and apply security patches in one place
  • Supports national crime statistics and policy analysis
  • Avoids data duplication — storage efficient

Disadvantages:

  • Single point of failure — if the central server fails, all forces lose access
  • Network dependency — remote access requires reliable high-speed connectivity
  • High-value target for hackers / cyberattacks; a breach exposes all data
  • Privacy concerns — GDPR compliance more complex across multiple jurisdictions
  • Bottleneck under high concurrent load from thousands of officers
For full marks: must include at least 2 advantages AND 2 disadvantages with development/explanation, not just listing.
3(c)(i)
Explain why data mining might be used to extract information from crime data held in a police database.
[4]
✓ Model Answer
  • Crime databases contain vast volumes of records — too large for manual analysis
  • Data mining uses algorithms to discover hidden patterns, correlations, and trends within the data
  • e.g. identifying crime hotspots — crimes cluster geographically or temporally in ways not obvious to human analysts
  • e.g. predicting repeat offender behaviour — patterns in previous offences predict future crimes
  • e.g. linking crimes from different precincts that share modus operandi / suspect descriptions
  • Helps allocate police resources efficiently (patrol times, locations)
  • Can reveal relationships between crimes and external factors (weather, events, social data)
Award marks for: identifying that manual analysis is insufficient + specific benefit with example.
3(c)(ii)
Explain, using an example, how data matching could be applied to the crime data.
[4]
✓ Model Answer
  • Data matching compares records across two or more datasets to identify records that refer to the same real-world entity
  • It can be exact matching (same ID) or fuzzy matching (similar names, addresses)

Example in crime context:

  • Matching a suspect's name/DOB in the crime database against the DVLA vehicle register to find their address
  • Matching witness phone numbers across crimes to identify the same witness appearing in multiple cases
  • Matching a description of stolen goods against pawnshop transaction records to trace receivers of stolen property
  • Cross-referencing fingerprint/DNA evidence records across crime scenes to link crimes to one perpetrator
Must include a clear example for full marks; general description alone capped at [2].
3(d)(i)
Outline two reasons why a password-only approach would not be secure.
[4]
✓ Model Answer — 2 marks per reason (identify + develop)
  • Passwords can be guessed / brute-forced: without lockout policies, an attacker can systematically try combinations; weak passwords are vulnerable to dictionary attacks
  • Passwords can be stolen / phished: users may be tricked into entering credentials on a fake login page; passwords can be intercepted if transmitted insecurely
  • Passwords can be shared: officers may share credentials, making it impossible to audit individual access and breaching the principle of least privilege
  • No verification of physical identity: a stolen password grants full access; multi-factor authentication (MFA) — e.g. biometrics, smart card — adds a second factor that is much harder to steal
3(d)(ii)
Identify three ways that legislation such as GDPR protects individuals whose data is stored electronically.
[3]
✓ Model Answer — 1 mark each
  • Right of access — individuals can request to see what data is held about them
  • Right to erasure ("right to be forgotten") — data must be deleted on request if no longer necessary
  • Data minimisation — only data necessary for the stated purpose may be collected/stored
  • Consent — explicit, informed consent must be obtained before collecting personal data
  • Data accuracy — organisations must keep data up to date and correct errors on request
  • Breach notification — individuals must be informed within 72 hours if their data is compromised
  • Right to portability — individuals can receive their data in a machine-readable format
Question 4 Network DB · OO DB · Data warehousing · Predictive modelling · Link analysis
4(a)
Identify two characteristics of a network database.
[2]
✓ Model Answer — 1 mark each
  • Data is organised as a graph of records (nodes) connected by pointers/links (edges) — not tables
  • Supports many-to-many relationships natively without junction tables
  • Data is accessed via navigational paths through the network (pointer chains)
  • Based on the CODASYL model; predates relational databases
  • A child record can have multiple parent records (unlike hierarchical databases)
  • Schema is defined at design time; changes are difficult (rigid structure)
4(b)
Discuss the advantages and disadvantages of an object-oriented database compared to a relational database.
[5]
✓ Model Answer — balanced discussion

Advantages of OO database:

  • Can store complex/multimedia data types (images, audio, video) that map poorly to relational tables
  • Objects encapsulate data + methods — behaviour is stored alongside data, reducing application code
  • Inheritance and polymorphism allow reuse and extension of data structures
  • Eliminates "impedance mismatch" — objects in code map directly to objects in the database
  • Better for complex hierarchical/nested data (e.g. crime evidence with sub-types)

Disadvantages of OO database:

  • Less mature technology — fewer standard query languages (no universal equivalent of SQL)
  • Fewer trained staff / less vendor support compared to relational DBMS
  • Existing police databases are relational — migration is expensive and risky
  • Ad-hoc querying is more difficult; reporting tools are less rich than for relational DBs
  • Optimisation / indexing less well understood at scale
4(c)
Outline two advantages of creating a data warehouse to store Europol crime data.
[4]
✓ Model Answer — 2 marks per advantage
  • Integrated historical data: consolidates data from multiple national police systems into one repository, enabling cross-border analysis over time that would be impossible from individual operational databases
  • Optimised for analytical queries: data warehouses use OLAP (Online Analytical Processing) structures optimised for complex read queries and aggregations, without degrading performance of live operational systems
  • Supports OLAP / multidimensional analysis: analysts can slice data by time, geography, crime type simultaneously — e.g. "thefts in border regions Q3 2022"
  • Separation from operational data: analytical workloads don't interfere with day-to-day police record updates; data is cleaned and standardised on import (ETL)
4(d)
Outline two benefits of applying predictive modelling to crime data.
[4]
✓ Model Answer — 2 marks per benefit
  • Proactive resource allocation: models predict where and when crimes are likely to occur, allowing police to pre-deploy patrols to high-risk areas before crimes happen, rather than only responding reactively
  • Identifying likely offenders / recidivism risk: models trained on offender histories can assess probability of reoffending, informing decisions about bail, parole, or targeted intervention programmes
  • Detecting emerging crime trends: statistical models identify deviations from baseline patterns early (e.g. sudden spike in organised retail theft) giving analysts actionable alerts
  • Cost efficiency: focusing investigative resources on predicted high-risk individuals/locations reduces wasted effort and can lower overall crime rates
4(e)
Explain how link analysis can be applied to crime datasets to investigate fraud, uncover terrorist networks and investigate criminal activities.
[5]
✓ Model Answer
  • Link analysis represents entities (people, organisations, locations, phone numbers, bank accounts) as nodes and relationships between them as edges in a graph
  • Algorithms traverse this graph to identify clusters of connected entities — revealing networks that are not apparent from individual records
  • Fraud: links between financial transactions, account holders, and IP addresses can reveal coordinated fraud rings — multiple fraudulent accounts connected through shared devices or money flows
  • Terrorist networks: communication records (calls, emails) link individuals; graph centrality metrics (e.g. betweenness centrality) identify key organisers/couriers even if they avoid direct communication with leaders
  • Criminal investigations: linking crime scenes by suspect description, vehicle registration, or witness appearances can connect seemingly unrelated crimes to one organised group
  • Can surface indirect connections: A → B → C where A and C are not directly linked but share common associates
Strong answer: defines nodes/edges, gives a specific application example, mentions what the analysis reveals that standard queries cannot.
