Rational Points on Quadratic Curves

A guided ascent through Legendre's Theorem and its proof
Stage 1

The Starting Question

We begin with a natural question: given a quadratic curve like 2x² + 7y² = 1, does it pass through any rational points — points (x, y) where both coordinates are rational numbers?

More generally, when does a curve of the form ax² + bxy + cy² = d contain rational points? We may assume a, b, c, d are integers (if they're rational, just multiply through by a common denominator).

The first simplification: eliminate the cross-term bxy. If a ≠ 0, complete the square in x to rewrite the left side as a(x + (b/2a)y)² + (c − b²/4a)y². Then substitute X = x + (b/2a)y, Y = y — this is invertible, so rational (x,y) correspond bijectively to rational (X,Y).

After this change of variables, any quadratic curve reduces to the diagonal form: aX² + c′Y² = d, with no cross-term.

Checkpoint 1
Why does completing the square preserve the existence of rational points? What property of the substitution X = x + (b/2a)y matters?
Exactly. The substitution x = X − (b/2a)Y, y = Y is linear with rational coefficients and invertible — so rational (x,y) ↔ rational (X,Y) one-to-one. The degree stays at 2; we've only removed the cross-term.
Not quite. The key is that X = x + (b/2a)y is an invertible linear map over ℚ. We can always undo it: x = X − (b/2a)Y. So rational solutions of one form correspond exactly to rational solutions of the other. Integer points aren't preserved in general — and the degree doesn't change.
Think about what "invertible over ℚ" means. Can you go back from (X, Y) to (x, y) using only rational arithmetic?
Stage 2

Homogenising: Two Variables to Three

So we've reduced to asking: does ax² + by² = c have rational solutions? Here's a clever move. Write rational x = X/Z and y = Y/Z for integers X, Y, Z. Then the equation becomes:

aX² + bY² = cZ²

We've gone from 2 rational unknowns to 3 integer unknowns. This looks worse — but the new equation is homogeneous (every term has degree 2). That means if (X, Y, Z) is a solution, so is (kX, kY, kZ) for any scalar k. This is a powerful structural property.

We exclude the trivial solution (0, 0, 0) and require Z ≠ 0 (to recover actual points on the original curve).

Checkpoint 2
Suppose (X, Y, Z) = (6, 10, 4) solves aX² + bY² = cZ². What simpler solution can you extract, and why?
Right. Homogeneity means (kX, kY, kZ) is a solution for any k — including k = 1/2. So (3, 5, 2) works too. We can always divide out common factors to get a primitive solution where gcd(X, Y, Z) = 1.
Since the equation is homogeneous, we can scale solutions freely. Dividing (6, 10, 4) by gcd = 2 gives (3, 5, 2), which is also a solution. You can't just set Z = 1 — that changes the equation's balance.
Stage 3

Stripping the Coefficients

Before tackling the hard question (when do solutions exist?), we simplify the coefficients of ax² + by² = cz². Three reductions:

1. Make each coefficient squarefree. If a = a′d² for d > 1, replace x with dx to absorb the square — the equation becomes a′x² + by² = cz² with smaller |a|. Repeat for b and c.

2. Remove common prime factors from all three. If p | a, b, c, divide through by p.

3. Remove common prime factors from any pair. If p | a and p | b (but p ∤ c), multiply the equation by p and absorb p into x and y. This decreases |abc| by a factor of p. Iterate until a, b, c are pairwise coprime.

After these reductions: a, b, c are squarefree and pairwise coprime. This is the "cleaned up" form that Legendre's Theorem addresses.
Checkpoint 3
Consider the equation 12x² + 18y² = 6z². After the full simplification procedure, what equation do we end up with?
That's it. 12 = 4·3, 18 = 2·9, 6 = 2·3. First divide all three by 6 → 2x² + 3y² = z². Now 2, 3, 1 are squarefree and pairwise coprime. Done.
Let's work through it: first divide 12, 18, 6 by their gcd (6) to get 2x² + 3y² = z². Check: 2, 3, 1 are squarefree and pairwise coprime — so we're done. The answer is 2x² + 3y² = z².
Start by dividing all three coefficients by their gcd. Then check if any coefficient still has a square factor.
Stage 4

Legendre's Theorem — The Statement

Now the main result. With a, b, c squarefree, pairwise coprime, and nonzero:

Theorem (Legendre). The equation ax² + by² = cz² has a nontrivial integer solution if and only if:

(i) ac is a square mod b,
(ii) bc is a square mod a,
(iii) −ab is a square mod c,
(iv) a and b do not both have the opposite sign from c.

Condition (iv) is just the sign constraint — if a, b > 0 and c < 0, the left side is positive and the right side is negative, so no solution. The deep content is in conditions (i)–(iii): three modular square conditions, one for each coefficient.

Note the symmetry: rewrite as ax² + by² + (−c)z² = 0, and the conditions become "−ac is a square mod b", "−bc is a square mod a", "ab is a square mod c", with no three coefficients sharing a sign. Each condition involves the other two coefficients, modulo the third.

Checkpoint 4
For the equation x² + y² = 3z², we have a=1, b=1, c=3. Check condition (iii): is −ab = −1 a square mod 3?
Correct. The squares mod 3 are 0² ≡ 0 and 1² ≡ 1 (and 2² ≡ 1), so the only quadratic residues mod 3 are {0, 1}. Since −1 ≡ 2 mod 3 is not among them, condition (iii) fails. This tells us x² + y² = 3z² has no nontrivial solution — which you can verify by trying small values!
Careful: we need to check whether 2 (since −1 ≡ 2 mod 3) is a quadratic residue mod 3. The squares mod 3 are: 0² = 0, 1² = 1, 2² = 4 ≡ 1. So the quadratic residues mod 3 are just {0, 1}. Since 2 ∉ {0,1}, condition (iii) fails. The equation has no nontrivial solution.
Stage 5

The Forward Direction — Why the Conditions Are Necessary

Suppose (x, y, z) ≠ (0,0,0) satisfies ax² + by² = cz². First, we may assume gcd(x,y) = gcd(x,z) = gcd(y,z) = 1 — any shared prime factor can be divided out (using squarefreeness of the coefficients to ensure this works cleanly).

Now, working mod b: the equation gives ax² ≡ cz² mod b. Multiply both sides by c:

acx² ≡ c²z² (mod b)

Since x and b are coprime (why? any prime dividing both x and b would force it to divide cz², hence c or z — contradicting coprimality of b,c and coprimality of x,z), x has a multiplicative inverse mod b. Multiplying by (x⁻¹)²:

ac ≡ (cz · x⁻¹)² (mod b)

So ac is a square mod b. The same argument, permuting roles, gives the other two conditions.

Checkpoint 5
In the step "any prime dividing both x and b would divide cz²" — why does this lead to a contradiction?
Exactly. Since p divides cz², either p | c or p | z (p is prime). But b and c are coprime, ruling out p | c. And if p | x and p | z, that contradicts pairwise coprimality of the solution. Both escape routes are blocked.
The argument is more specific: if p | b and p | cz², then (since p is prime) p | c or p | z. The first contradicts gcd(b,c) = 1; the second contradicts gcd(x,z) = 1 (since we assumed p | x too). Two separate coprimality conditions each block one possibility.
Stage 6

The Converse Strategy — Factoring mod abc

The converse is the hard part: assuming the three quadratic residue conditions, construct a solution. The strategy is beautiful.

If we could factor ax² + by² + cz² = (a₁x + b₁y + c₁z)(a₂x + b₂y + c₂z) over ℤ, we'd just solve one linear factor = 0. This almost never works over ℤ. But the quadratic residue conditions guarantee a factorisation modulo abc.

Here's the idea for the mod-a piece. We need to factor by² + cz² mod a. Multiply by b to get b²y² + bcz². Since −bc ≡ r² mod a for some r (this is one of our conditions!), we get b²y² − r²z² ≡ (by + rz)(by − rz) mod a. Dividing by b (which is coprime to a, hence invertible mod a) gives the factorisation.

Do the same mod b and mod c, then glue the three factorisations together using the Chinese Remainder Theorem to get a single factorisation mod abc.

Checkpoint 6
Why can we "divide by b" when factoring mod a? What algebraic property is being used?
Exactly right. In modular arithmetic, "dividing by b mod a" means multiplying by b⁻¹ mod a. This inverse exists precisely because gcd(a,b) = 1 — which we arranged in the coefficient simplification stage. Every reduction we made earlier is paying off here.
We're working in ℤ/aℤ, not in ℚ. Division in modular arithmetic requires a multiplicative inverse: b⁻¹ mod a exists if and only if gcd(a,b) = 1. Since a and b are coprime (we made sure of that!), b has an inverse mod a.
Stage 7

The Pigeonhole Step — Bounding the Solution

We now have a linear congruence a₁x + b₁y + c₁z ≡ 0 mod abc coming from the factorisation. We need an actual integer solution where ax² + by² + cz² is small.

Consider all integer triples (x, y, z) with 0 ≤ x < α, 0 ≤ y < β, 0 ≤ z < γ. The number of such triples is λ(α)λ(β)λ(γ), where λ(t) counts integers in [0, t). If t is not an integer, λ(t) = 1 + ⌊t⌋ > t.

If λ(α)λ(β)λ(γ) > |abc|, the Pigeonhole Principle guarantees two triples give the same residue mod abc for the linear form. Their difference (x, y, z) ≠ (0,0,0) satisfies our congruence, with |x| < α, |y| < β, |z| < γ.

Choose α = √|bc|, β = √|ac|, γ = √|ab|. Then αβγ = |abc|, and the bound gives:

−|abc| < ax² + by² + cz² < 2|abc|

Combined with ax² + by² + cz² ≡ 0 mod |abc|, we get either ax² + by² + cz² = 0 (done!) or ax² + by² + cz² = ±abc.

Checkpoint 7
The pigeonhole argument requires λ(α)λ(β)λ(γ) > αβγ = |abc|. When does λ(t) > t hold?
Precisely. If t is an integer, λ(t) = t (the integers 0, 1, …, t−1). If t is not an integer, we get the "bonus": λ(t) = ⌊t⌋ + 1 > t. So we need at least one of α, β, γ to be non-integer. Since a, b, c are coprime and squarefree, √|bc| etc. are integers only when a, b, c are all ±1 — which gives the trivial case x² + y² = z².
Not quite. λ(t) counts integers in [0, t). If t is an integer, there are exactly t such integers, so λ(t) = t. If t is not an integer, then λ(t) = ⌊t⌋ + 1 > t — we get one extra integer. The condition "t is irrational" is stronger than needed; any non-integer suffices.
Stage 8

The Final Trick — From ±abc to Zero

If we're unlucky, the pigeonhole step gives ax² + by² + cz² = −abc rather than 0 (rewriting with the sign convention a, b > 0, c < 0). We need to convert this into a genuine solution of ax² + by² + cz² = 0.

Rewrite ax² + by² + cz² = −abc as ax² + by² + c(z² + ab) = 0. We want aX² + bY² + cZ² = 0 with Z² = (z² + ab)². So multiply ax² + by² by z² + ab:

(ax² + by²)(z² + ab) = ax²z² + a²bx² + by²z² + ab²y²
    = a(xz + by)² + b(yz − ax)²

This is a Brahmagupta–style identity: products of sums-of-forms are themselves sums-of-forms. Setting X = xz + by, Y = yz − ax, Z = z² + ab gives a genuine solution, and Z > 0 ensures it's nontrivial.

Checkpoint 8
Why is the new solution nontrivial? Specifically, why is Z = z² + ab > 0?
Right — and both options B and C capture this. Since we arranged a, b > 0 (and c < 0), the product ab is strictly positive. Adding z² ≥ 0 gives Z = z² + ab ≥ ab > 0. So (X, Y, Z) can't be (0, 0, 0).
The core point: we arranged a, b > 0 in the sign convention step, so ab > 0. Since z² ≥ 0, we get Z = z² + ab ≥ ab > 0 unconditionally — no assumption on z is needed beyond z being an integer. Option A is too strong (z could be 0).
Stage 9

Applying the Theorem — Quadratic Residue Checks

To use Legendre's Theorem in practice, you need to determine which numbers are squares mod n. For small n, brute force: compute k² mod n for k = 0, 1, …, ⌊n/2⌋.

For example, squares mod 5: 0² = 0, 1² = 1, 2² = 4. So the quadratic residues mod 5 are {0, 1, 4}. The non-residues are {2, 3}.

Checkpoint 9
Does the equation 2x² + 3y² = 5z² have a nontrivial integer solution? Check all three Legendre conditions (a=2, b=3, c=5).
Yes! Let's verify: (i) ac = 10 mod 3: 10 ≡ 1 mod 3, and 1 is a square mod 3. ✓ (ii) bc = 15 mod 2: 15 ≡ 1 mod 2, and 1 is a square mod 2. ✓ (iii) −ab = −6 mod 5: −6 ≡ 4 mod 5, and 4 = 2² is a square mod 5. ✓ (iv) a=2, b=3 are positive and c=5 is positive — consistent signs. ✓ All conditions met! Indeed (1, 1, 1) is a solution.
Let's check: (i) ac = 10 ≡ 1 mod 3 — yes, 1 is a square. (ii) bc = 15 ≡ 1 mod 2 — yes, 1 is a square. (iii) −ab = −6 ≡ 4 mod 5 — yes, 4 = 2². (iv) Signs are all positive — fine. All conditions pass, so yes, solutions exist. In fact (1, 1, 1) works: 2 + 3 = 5.
Compute ac mod b, bc mod a, and −ab mod c. Check each against the quadratic residues of the respective modulus. Also check signs.
Stage 10

The Big Picture

Let's trace the full logical arc:

Reduce: complete the square → diagonal form → homogenise → strip square factors → make coefficients coprime. Each step is reversible or equivalent for the existence question.

Characterise: Legendre's Theorem gives a clean, checkable criterion — three modular conditions plus a sign condition.

Prove: the forward direction uses coprimality to extract quadratic residue information. The converse factors the form mod abc (using the residue conditions), applies pigeonhole to get a small solution, and uses an algebraic identity to reach exact zero.

This is a prototype of a local-to-global principle: conditions modulo each prime (the quadratic residue checks) and a sign condition (the "real place") together determine a global arithmetic fact. This idea matures into the Hasse–Minkowski theorem for quadratic forms.
Final Checkpoint
Which tool from the proof specifically required the coefficients to be pairwise coprime (not just squarefree)?
Exactly. CRT requires the moduli to be pairwise coprime. Without that, we can't combine three separate factorisations (mod a, mod b, mod c) into a single factorisation mod abc. Every simplification step at the beginning was setting up this moment.
The Chinese Remainder Theorem is the step that directly needs pairwise coprimality. CRT says: if gcd(a,b) = gcd(a,c) = gcd(b,c) = 1, then systems of congruences mod a, b, c can be combined into one mod abc. This is exactly how the three separate factorisations get glued together.
