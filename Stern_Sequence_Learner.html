The Stern Sequence

Stern's Diatomic Sequence · Calkin-Wilf · Enumerating ℚ
A simple recurrence that, remarkably, enumerates every positive rational number exactly once.

The Recurrence

Stern's diatomic sequence is built from two rules: even-indexed terms copy, odd-indexed terms sum. That's it. Everything else follows.

S(0) = 0,   S(1) = 1
S(2n) = S(n)     (even indices: just copy)
S(2n+1) = S(n) + S(n+1)     (odd indices: sum neighbours)

The first terms

Building it by hand

Start with S(0)=0 and S(1)=1. Then:

S(2) = S(1) = 1  (even rule). S(3) = S(1) + S(2) = 1 + 1 = 2  (odd rule). S(4) = S(2) = 1. S(5) = S(2) + S(3) = 1 + 2 = 3. And so on. The even rule halves the index and looks up; the odd rule sums two adjacent earlier values.

The spreadsheet trick

Your slides show how to implement this in a spreadsheet using INDIRECT. Column B stores ⌊n/2⌋ (the "row" to look up), column C stores the target row reference, and column D uses =INDIRECT("D" & C_n) + INDIRECT("D" & C_{n+1}) to compute the odd-index sums. The INDIRECT function lets a formula reference cells dynamically, which is exactly what a recurrence needs.

Enumerating ℚ

The sequence of rationals S(n)/S(n+1) produces every positive rational number exactly once. This is the Calkin-Wilf enumeration: a bijection from ℕ to ℚ+.

Look at the first few: 0/1, 1/1, 1/2, 2/1, 1/3, 3/2, 2/3, 3/1, 1/4, 4/3, 3/5, 5/2, … Every positive fraction appears, already in lowest terms (because consecutive Stern values are always coprime), and no fraction ever repeats.

Key Properties

Click each card to see the proof sketch or deeper explanation.

Consecutive coprimality

gcd(S(n), S(n+1)) = 1 for all n ≥ 1.

▸ click for proof

By strong induction. Base: gcd(S(1), S(2)) = gcd(1,1) = 1. For S(2n) and S(2n+1): gcd(S(n), S(n)+S(n+1)) = gcd(S(n), S(n+1)) = 1 by induction. For S(2n+1) and S(2n+2): gcd(S(n)+S(n+1), S(n+1)) = gcd(S(n), S(n+1)) = 1. The even rule preserves coprimality because it doesn't change values; the odd rule adds coprime neighbours, which stays coprime with either one.

Every positive rational appears

The Stern-Brocot / Calkin-Wilf bijection ℕ → ℚ+.

▸ click for why

Consider the map f(p/q): if p/q < 1, map to p/(q−p); if p/q > 1, map to (p−q)/q; if p/q = 1, stop. This is essentially the Euclidean algorithm, and it always terminates at 1/1. Reversing this process generates the Calkin-Wilf tree, whose breadth-first reading is exactly the sequence S(n)/S(n+1). Since the Euclidean algorithm reaches every reduced fraction, the tree (and hence the sequence) enumerates all of ℚ+.

Self-similarity

The sequence has fractal structure: S restricted to [2k, 2k+1] looks like a scaled copy.

▸ click for more

Because S(2n) = S(n), even-indexed values are a literal copy of the sequence at half scale. The odd-indexed values interleave sums. Each "generation" (between successive powers of 2) adds one layer to the Stern-Brocot tree. The maximum of S in the range [0, 2k] is the (k+1)-th Fibonacci number, connecting Stern to Fibonacci growth.

Sum over rows

∑ S(n) for 2k ≤ n < 2k+1 = 3k + 1.

▸ click for more

Row k of the sequence (between powers of 2) has 2k elements. The sum follows from the recurrence: each row's sum is three times the previous row's sum minus one correction. This connects to the fact that each generation of the Stern-Brocot tree has a tripling structure.

The Calkin-Wilf Tree

Every positive rational appears exactly once as a node. The root is 1/1. The children of p/q are p/(p+q) on the left and (p+q)/q on the right. Reading the tree breadth-first gives the Stern-Brocot enumeration.

Python Implementation

Three lines for the recurrence. The rest is exploration.

def stern(n): # Compute S(n) via the recurrence if n == 0: return 0 if n == 1: return 1 if n % 2 == 0: return stern(n // 2) # even: S(2n) = S(n) else: k = n // 2 return stern(k) + stern(k + 1) # odd: S(2n+1) = S(n) + S(n+1) # Build a table iteratively (much faster for many terms) def stern_table(N): s = [0, 1] for i in range(2, N): if i % 2 == 0: s.append(s[i // 2]) else: s.append(s[i // 2] + s[i // 2 + 1]) return s # Enumerate rationals s = stern_table(100) rationals = [(s[n], s[n+1]) for n in range(1, 99)] print(rationals[:15]) # [(1,1),(1,2),(2,1),(1,3),(3,2),(2,3),(3,1),(1,4),(4,3),...]

The recursive version mirrors the mathematical definition directly. For large N, the iterative stern_table avoids repeated computation. The key insight: the iterative version works because each S(n) depends only on terms with index ≤ n/2 + 1, so a single forward pass suffices.
