Non-Repeating Patterns

From genetic codons to De Bruijn sequences: why maximal information
demands that every subsequence appears exactly once

I. Wot Is Life Anyhow?

A crystal repeats. 0101010101... is perfectly ordered, perfectly boring, and carries almost no information. You can predict every next bit. Life, by contrast, is a non-repeating pattern of high information content: a genome whose four-letter alphabet (A, T, G, C) encodes instructions that never quite settle into periodicity.

Why does repetition kill information?

Consider two binary strings of length 20:

01010101010101010101 — period 2, predictable after 2 bits.

00010011010111100001 — every 4-tuple appears, nothing is predictable.

Shannon's insight: the information content of a symbol depends on how surprising it is. If you can predict the next bit, it carries zero information. A string where every n-tuple substring appears exactly once is maximally informative for that window size: you can never predict what comes next from any local context alone.

DNA as a quaternary code

DNA uses four nucleotides rather than two bits, giving a quaternary (base-4) alphabet. A codon is a 3-tuple of this alphabet, yielding 43 = 64 possible codons that map to 20 amino acids plus stop signals. The genetic code is therefore a windowed read of a long non-repeating string, exactly the structure we are about to explore in binary.

The key analogy: a reading frame slides along the DNA strand one position at a time, extracting overlapping n-tuples, just as we extract overlapping binary substrings from a De Bruijn sequence.

II. Shannon Entropy and Surprise

Claude Shannon defined the entropy of a discrete random variable X as:

H(X) = −∑ p(x) log2 p(x)    bits

For a fair binary source, each bit carries exactly 1 bit of entropy. But when we ask about n-tuple substrings, the question becomes richer: how many distinct patterns of length n can a string contain before it must repeat?

Maximal entropy and the De Bruijn bound

There are 2n distinct binary strings of length n. A binary string of length L contains (L − n + 1) overlapping n-tuples. For every n-tuple to appear exactly once, we need L − n + 1 = 2n, giving:

L = 2n + n − 1

This is the length of a De Bruijn sequence B(2, n). When every n-tuple appears with equal frequency (once), the empirical entropy of the n-tuple distribution is maximised at exactly n bits. The sequence is a maximum entropy object for its window size.

Hypothesis: what happens if we try to exceed this bound?

Suppose you have placed all 2n distinct n-tuples. Any further bit creates a new n-tuple from the last (n−1) bits plus the new bit. But all 2n possibilities are exhausted: the new n-tuple must be a repeat. The pigeonhole principle prevents extension. The De Bruijn length is therefore both the maximum and the minimum for a complete catalogue.

III. De Bruijn Sequences: The Construction

A De Bruijn sequence B(k, n) over an alphabet of size k is a cyclic sequence in which every possible string of length n appears exactly once as a contiguous substring. For binary (k=2), the linear (non-cyclic) form appends (n−1) bits to close the cycle, giving a total length of 2n + n − 1.

The graph-theoretic view: Eulerian circuits on De Bruijn graphs

Build a directed graph whose nodes are all binary strings of length (n−1). Draw an edge from node u to node v whenever the last (n−2) characters of u match the first (n−2) characters of v, labelling the edge with the n-character string formed by appending v's last character to u. This graph has exactly 2n edges. A De Bruijn sequence corresponds to an Eulerian circuit: a path that traverses every edge exactly once.

The number of distinct De Bruijn sequences B(2, n) grows super-exponentially. For n=3 there are 2, for n=4 there are 16, and the count is given by the BEST theorem (de Bruijn, Ehrenfest, Smith, Tutte).

Martin's algorithm (greedy/prefer-zero)

A simple greedy strategy works for manual exploration: at each step, try appending 0; if that creates a repeated n-tuple, try 1. But pure greedy can hit dead ends. The mathematically guaranteed construction uses Lyndon word concatenation: enumerate all binary Lyndon words of lengths dividing n in lexicographic order, concatenate them, and wrap cyclically.

def debruijn(k, n):
    """Lyndon word concatenation for B(k, n)."""
    a = [0] * (k * n)
    sequence = []

    def db(t, p):
        if t > n:
            if n % p == 0:
                sequence.extend(a[1:p+1])
        else:
            a[t] = a[t - p]
            db(t + 1, p)
            for j in range(a[t - p] + 1, k):
                a[t] = j
                db(t + 1, t)

    db(1, 1)
    return sequence  # cyclic; append [:n-1] for linear

# Example: linear B(2, 4)
cyc = debruijn(2, 4)
linear = cyc + cyc[:3]
print(''.join(map(str, linear)))
# -> 0000100110101111000

V. Entropy Analysis

Why does a complete De Bruijn sequence maximise entropy?

In a complete B(2, n), every n-tuple appears exactly once, so each has probability p = 1/2n. The entropy is:

H = −2n × (1/2n) × log2(1/2n) = n bits

This is the theoretical maximum for an n-bit alphabet. Any repetition would increase some tuple's count at the expense of a missing tuple, reducing entropy below n. The De Bruijn sequence is therefore the maximum entropy binary string for window size n.

Partial sequences and approach to maximum entropy

As you build the sequence in the explorer, watch the live entropy climb. Each new unique tuple increases the count of distinct symbols in the distribution. The entropy rises steeply at first (new tuples are cheap) and saturates as you approach the target. A dead-end path will have lower entropy than a complete solution, revealing that the greedy path was locally optimal but globally suboptimal.

VI. Checksums and Verification

The PDF's spreadsheets use a neat verification trick: if every n-tuple from 000...0 to 111...1 appears exactly once, then the sum of their decimal values equals a known target. For an n-tuple, the integers 0 through 2n−1 each appear once, so:

Checksum = ∑k=02n−1 k = 2n(2n − 1) / 2

Beyond binary: quaternary checksums and codon space

For DNA's 4-letter alphabet with codon length 3, there are 43 = 64 codons. A De Bruijn sequence B(4, 3) would have length 43 + 2 = 66 bases and contain every possible codon exactly once. The quaternary checksum would be ∑k=063 k = 2016. Real genomes are far longer than 66 bases, so codons repeat abundantly, but the information-theoretic principle remains: non-repetition at a given window size characterises maximal local information density.
