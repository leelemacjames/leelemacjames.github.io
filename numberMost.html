Non-Repeating Patterns

From genetic codons to De Bruijn sequences: why maximal information
demands that every subsequence appears exactly once

I. Wot Is Life Anyhow?

A crystal repeats. 0101010101... is perfectly ordered, perfectly boring, and carries almost no information. You can predict every next bit. Life, by contrast, is a non-repeating pattern of high information content: a genome whose four-letter alphabet (A, T, G, C) encodes instructions that never quite settle into periodicity.

Why does repetition kill information?

Consider two binary strings of length 20:

01010101010101010101 — period 2, predictable after 2 bits.

00010011010111100001 — every 4-tuple appears, nothing is predictable.

Shannon's insight: the information content of a symbol depends on how surprising it is. If you can predict the next bit, it carries zero information. A string where every n-tuple substring appears exactly once is maximally informative for that window size: you can never predict what comes next from any local context alone.

DNA as a quaternary code — and the ternary stepping stone

DNA uses four nucleotides (A, T, G, C), giving a quaternary alphabet. A codon is a 3-tuple, yielding 43 = 64 possible codons mapping to 20 amino acids plus stop signals. The genetic code is a windowed read of a long string, exactly the structure we explore here.

The jump from binary (k=2) to quaternary (k=4) is large. Ternary (k=3) provides a natural intermediate: three symbols give 3n possible n-tuples, and the De Bruijn sequence B(3, n) has length 3n + n − 1. Use the toggle in the explorer below to switch between binary and ternary and watch how the combinatorial space grows. Ternary 3-tuples (27 words, length 29) sit neatly between binary 4-tuples (16 words, length 19) and binary 5-tuples (32 words, length 36), making the comparison illuminating.

II. Shannon Entropy and Surprise

Claude Shannon defined the entropy of a discrete random variable X as:

H(X) = −∑ p(x) log2 p(x)    bits

For a fair binary source, each bit carries exactly 1 bit of entropy. But when we ask about n-tuple substrings, the question becomes richer: how many distinct patterns of length n can a string contain before it must repeat?

Maximal entropy and the De Bruijn bound

Over an alphabet of size k, there are kn distinct strings of length n. A sequence of length L contains (L − n + 1) overlapping n-tuples. For every n-tuple to appear exactly once:

L = kn + n − 1

This is the linear length of a De Bruijn sequence B(k, n). When every n-tuple appears with equal frequency (once), the empirical entropy of the n-tuple distribution is maximised at log2(kn) = n log2 k bits. For binary this is n bits; for ternary, n log2 3 ≈ 1.585n bits.

Hypothesis: what happens if we try to exceed this bound?

Suppose you have placed all kn distinct n-tuples. Any further symbol creates a new n-tuple from the last (n−1) symbols plus the new one. But all kn possibilities are exhausted: the new n-tuple must be a repeat. The pigeonhole principle prevents extension. The De Bruijn length is therefore both the maximum and the minimum for a complete catalogue.

III. De Bruijn Sequences: The Construction

A De Bruijn sequence B(k, n) over an alphabet of size k is a cyclic sequence in which every possible string of length n appears exactly once as a contiguous substring. The linear form appends (n−1) symbols to close the cycle, giving total length kn + n − 1.

The graph-theoretic view: Eulerian circuits on De Bruijn graphs

Build a directed graph whose nodes are all strings of length (n−1) over the k-symbol alphabet. Draw an edge from node u to node v whenever u's last (n−2) characters match v's first (n−2) characters, labelling the edge with the n-character string formed by appending v's last character to u. This graph has exactly kn edges (each node has out-degree k). A De Bruijn sequence corresponds to an Eulerian circuit: a path traversing every edge exactly once.

For ternary, each node has out-degree 3 instead of 2, and the graph is denser. The number of distinct De Bruijn sequences is given by the BEST theorem and grows super-exponentially in both k and n.

Lyndon word concatenation (works for any k)

The algorithm below produces a valid B(k, n) for any alphabet size k. It enumerates Lyndon words whose lengths divide n in lexicographic order and concatenates them. This is O(kn) and guaranteed to produce a valid sequence.

def debruijn(k, n):
    """Lyndon word concatenation for B(k, n)."""
    a = [0] * (k * n)
    sequence = []

    def db(t, p):
        if t > n:
            if n % p == 0:
                sequence.extend(a[1:p+1])
        else:
            a[t] = a[t - p]
            db(t + 1, p)
            for j in range(a[t - p] + 1, k):
                a[t] = j
                db(t + 1, t)

    db(1, 1)
    return sequence  # cyclic; append [:n-1] for linear

# Binary B(2, 4): length 19
cyc = debruijn(2, 4)
print(''.join(map(str, cyc + cyc[:3])))

# Ternary B(3, 3): length 29
cyc = debruijn(3, 3)
print(''.join(map(str, cyc + cyc[:2])))

V. Entropy Analysis

Why does a complete De Bruijn sequence maximise entropy?

In a complete B(k, n), every n-tuple appears exactly once, so each has probability p = 1/kn. The entropy is:

H = −kn × (1/kn) × log2(1/kn) = n log2 k   bits

For binary (k=2) this gives n bits. For ternary (k=3) it gives n × log2 3 ≈ 1.585n bits. This is the theoretical maximum for an n-window over a k-symbol alphabet. Any repetition reduces entropy below this ceiling.

Comparing binary and ternary growth

The sequence length grows as kn + n − 1, so ternary sequences are dramatically longer for the same n. For n=4: binary needs 19 symbols (16 tuples), ternary needs 84 (81 tuples). For n=5: binary gives 36 symbols, ternary gives 248. This mirrors the jump from binary computing to the richer alphabets found in biology: more symbols per position means exponentially more combinatorial possibilities per window.

Beyond ternary: quaternary checksums and codon space

For DNA's 4-letter alphabet with codon length 3, there are 43 = 64 codons. A De Bruijn sequence B(4, 3) would have length 66 bases containing every possible codon exactly once, with checksum 2016. Real genomes are far longer, so codons repeat abundantly, but the principle stands: non-repetition at a given window size characterises maximal local information density.

The ternary case (k=3) in the explorer above is the nearest tractable stepping stone to this biological reality, bridging the gap between the binary toy model and the full quaternary codon space.
