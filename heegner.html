Prime-Rich Quadratics& the Heegner Numbers

How nine special discriminants produce extraordinary runs of primes — and the surprising patterns that emerge after the run ends.

The Setup

In 1772, Euler noticed something remarkable: the polynomial x² + x + 41 produces prime numbers for every integer input from 0 to 39. That's forty consecutive primes — a feat no other quadratic can match.

This isn't a coincidence. It's a consequence of the fact that −163, the discriminant of the associated quadratic form x² + xy + 41y², has class number 1. But what does that actually mean?

A quadratic form like ax² + bxy + cy² is a machine for representing integers — you feed in pairs of integers (x, y) and it outputs a number. The discriminant Δ = b² − 4ac is a fixed fingerprint of the form's geometry. Forms with the same discriminant are grouped together, and within each group some forms can be transformed into each other by a change of variables (an equivalence). The class number counts how many genuinely distinct forms share a given discriminant — how many non-equivalent "machines" there are.

Class number 1 means: there's only one machine. And that uniqueness is what keeps the primes flowing. Two ways to see why:

📮 One Post Office vs. Two

Think of each quadratic form as a post office, and each integer as a parcel. "Representing" a number means that post office can handle that parcel — there exist integers x, y you can plug in to produce it.

When the class number is 1, there's one post office in town. Every number that can be represented at all gets routed to the same place — the principal form. No sorting, no ambiguity, no competing offices. The arithmetic is completely clean.

When the class number is bigger than 1, you have rival post offices. Some parcels go to one, some to the other, and the question "which form represents n?" becomes a genuine sorting problem. That sorting creates opportunities for the polynomial to factor early — there are now multiple ways the arithmetic can "split," and composites creep in sooner.

Class number 1 means: no forks. One road, one destination. The polynomial stays prime because there's nowhere else for the arithmetic to go.

🔢 Δ = −163 (class 1) vs. Δ = −20 (class 2)

For Δ = −163, Hatcher's reduction algorithm asks: what odd values of h satisfy h² ≤ 163/3 ≈ 54.3? That gives h = 1, 3, 5, 7. For each, compute ac = (163 + h²)/4, getting 41, 43, 47, 53 — all prime. Since a ≤ c, the only option each time is a = 1. But h ≤ a forces h = 1 in every case. All roads collapse to the single form x² + xy + 41y². One class. Done.

Δ = −163 · Class number 1

Only form: x² + xy + 41y²

Polynomial x² + x + 41 stays prime for x = 0 to 39. 40 consecutive primes.

Δ = −20 · Class number 2

Form 1: x² + 5y²

Form 2: 2x² + 2xy + 3y²

Polynomial x² + x + 5 fails at x = 5, giving 35 = 5 × 7. Only 5 primes.

The second form 2x² + 2xy + 3y² genuinely exists: it has discriminant 4 − 24 = −20, it's primitive (gcd of coefficients is 1), and no change of variables can turn it into x² + 5y². Its mere existence — a second equivalence class — is what breaks the polynomial's prime streak early. The number 3, for instance, is represented by the second form (set x = 0, y = 1) but not by the principal form. The arithmetic has a fork in the road, and forks kill primes.

The negative fundamental discriminants with class number 1 are precisely: −3, −4, −7, −8, −11, −19, −43, −67, −163.

These correspond to the Heegner numbers: 1, 2, 3, 7, 11, 19, 43, 67, 163. For each, the polynomial x² + x + p where p = (D+1)/4 (for D ≡ 3 mod 4) generates primes for x = 0 through p − 2.

The class number 1 condition guarantees that the only way the polynomial can produce a composite value in its "prime range" would require a factorisation that contradicts the uniqueness of the quadratic form. Once x reaches p − 1, the polynomial evaluates to p², and the spell breaks.

After the Breakdown

The truly surprising thing isn't that the primes eventually stop — it's what happens after they stop. Take x² + x + 41 again. At x = 40 we get 41² = 1681 (composite). At x = 41 we get 1763 = 41 × 43. Then two primes, then 2021 = 43 × 47. Then four primes, then 2491 = 47 × 53. The composites are products of consecutive values from the original prime sequence, and the gaps between composites grow as 2, 4, 6, 8, 10 primes — mirroring the original staircase.

The Identity Behind the Pattern

For P(x) = x² + x + 41, there's a beautiful algebraic identity: P(40 + n²) = P(n − 1) · P(n). So at n = 1 the composite lands at x = 41, at n = 2 it lands at x = 44, at n = 3 it's x = 49, and so on. Since the gaps between consecutive squares grow as 2n + 1, the prime runs between composites grow in step: 2, 4, 6, 8, …

This pattern eventually breaks down visually at x = 81 for the D = 163 case, where we get 41 × 163. The algebraic identity itself does not break; what changes is that extra off-lattice composites begin to appear between the square-indexed landmarks.

Post-Breakdown Pattern

Does This Apply to All Heegner Quadratics?

Yes — and this is the key insight. Each polynomial P(x) = x² + x + p (where p = (D+1)/4) satisfies an analogous identity: P((p−1) + n²) = P(n−1) · P(n). The constant p plays the same structural role as 41 does for D = 163. So every Heegner quadratic exhibits the same post-breakdown choreography of composite = (consecutive pair product) interspersed with growing prime runs.

The main difference is how long the post-breakdown regularity lasts. For D = 163 the identity continues indefinitely, but the clean staircase of prime-only gaps is disrupted once additional composites appear between the lattice points. For smaller discriminants the same thing happens earlier.

Lattice vs. Off-Lattice Composites

For P(x)=x²+x+41, the forced composites sit on the square lattice x=40+n². These are the values dictated by the identity P(40+n²)=P(n−1)P(n). But once x reaches 81, new composites appear away from that lattice. The square skeleton is still there; it is simply no longer alone.

Lattice composites

41, 44, 49, 56, 65, 76, 89, 104, 121, 140, 161, …

These occur exactly at x=40+n² and factor as consecutive terms from the original prime sequence.

Off-lattice breakers

81, 82, 84, 87, 91, 96, 102, 109, 117, 122, 123, 126, 127, …

These are the extra composites that interrupt the clean arithmetic growth of the prime runs.

Up to x=76, every composite is on the lattice. At x=81, the first breaker appears. After that, the lattice points continue to obey their odd-gap law 3,5,7,9,11,13,…, but off-lattice composites begin filling the spaces between them. The staircase remains real; it just stops being visually dominant.

Key distinction: the identity never fails. What fails is the earlier illusion that the only composites are the lattice composites. Once extra factorisations appear, the clean prime runs are overwritten by a denser composite landscape.

Why It Matters

The class number 1 problem — determining exactly which discriminants have this property — was conjectured by Gauss around 1800 and fully resolved only in the 1950s–60s by Heegner, Baker, and Stark. The nine negative fundamental discriminants listed above are provably all there are. This finiteness result is itself remarkable: for positive discriminants, it's conjectured (but still unproven) that infinitely many have class number 1.

The prime-generating polynomials are a visible shadow of this deep arithmetic uniqueness. When there's only one equivalence class of forms, the polynomial has no room to factor prematurely. The post-breakdown patterns are an algebraic afterglow — the identity P(c+n²)=P(n−1)·P(n) is purely algebraic, but the question of whether the intervening values are prime dips back into analytic number theory.

It's also worth noting the curious fact, mentioned by Hatcher, that the combined prime values from all the nontrivial class-1 polynomials account for every prime below 100 except 79.
