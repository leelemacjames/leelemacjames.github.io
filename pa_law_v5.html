Isoperimetric

The isoperimetric ratio
B²/A on a lattice

This learner is a companion to the interactive P/A explorer, which introduced Pick's theorem on the square and Eisenstein lattices. That app showed B/A computed for polygons you draw. This learner asks a sharper question: is B/A even the right thing to measure?

The classical isoperimetric problem asks: among all shapes with a given perimeter, which encloses the most area? The answer is the circle. But the quantity that captures shape efficiency — independent of size — is not P/A.

Why P/A fails as a measure of shape

Dilate any shape by a factor r. Its perimeter scales as rP, its area as r²A:

P/A → rP/r²A = (1/r)·(P/A) → 0 as r → ∞

So P/A → 0 for any shape under dilation — a square, a circle, a fractal coastline. It carries no shape information whatsoever, only size. On a lattice, Pick confirms this directly: for an n×n square, B = 4n and A = n², so B/A = 4/n → 0.

The scale-invariant measure

Dilation by r gives (rP)²/(r²A) = P²/A unchanged. So P²/A is scale-invariant — it measures shape efficiency independent of size:

P²/A — the isoperimetric ratio — is invariant under dilation
The isoperimetric inequality: for any shape, P²/A ≥ 4π, with equality only for the circle. On a lattice, P is replaced by the boundary count B and A is given by Pick. The question becomes: what is the minimum B²/A achievable on each lattice, and how does the basis determinant Δ enter?
Pick + scaling

Pick's theorem
and scale invariance

Pick's theorem is A = Δ(I + B/2 − 1). The correct asymptotic behaviour under dilation is B/A → 0. The scale-invariant quantity is B²/A, which Pick lets us express purely in terms of lattice point counts.

What Pick says about B/A

From Pick exactly:

B/A = (2/Δ) · (1 − ΔI/A + Δ/A)

For a large well-filled polygon, I grows like A/Δ, so ΔI/A → 1 and Δ/A → 0. The bracket → 1 − 1 + 0 = 0. Therefore:

B/A → 0 for any large filled polygon — trivially, by dilation

This is not an interesting limit. An n×n square has B = 4n, A = n², so B/A = 4/n → 0 regardless of shape.

The correct scale-invariant quantity

B²/A is invariant under lattice dilation by n (all coordinates scaled by n): B → nB, A → n²A, so B²/A → n²B²/n²A = B²/A. Using Pick:

B²/A = B² / [Δ(I + B/2 − 1)]

This is the lattice analogue of the classical isoperimetric ratio P²/A. It is dimensionless in lattice units and measures shape efficiency independent of size.

The n×n square — explicit check

For a square n×n on the square lattice (Δ=1): A = n², B = 4n, I = (n−1)².

B²/A = 16n²/n² = 16 — constant for all n ✓

And Pick confirms: A = (n−1)² + 4n/2 − 1 = n² ✓. The ratio 16 matches the continuous square (P²/A = (4L)²/L² = 16). Scale invariance is exact.

The role of Δ: on the Eisenstein lattice, the same n×n parallelogram (in oblique coordinates) has area Δ·n² = (√3/2)n² and boundary 4n (same count). So B²/A = 16n²/((√3/2)n²) = 32/√3 ≈ 18.48. The lattice determinant enters the denominator — a denser lattice (smaller Δ) gives a larger B²/A for the same combinatorial shape, because the same boundary count encloses less Euclidean area.

Isoperimetric inequality

Lattice isoperimetric
B²/A ≥ 4π via Pick

The continuous isoperimetric inequality P²/A ≥ 4π (with equality for the circle) has a lattice analogue. Pick's theorem is the bridge.

The inequality

For any lattice polygon with boundary count B, interior count I, and fundamental domain area Δ:

B²/A ≥ 4π — approached as the polygon approximates a circle

The minimum is achieved by the most circle-like convex lattice polygon available — which depends on how many boundary directions the lattice permits.

How Δ enters

Since A = Δ(I + B/2 − 1), the ratio B²/A can be written:

B²/A = B² / [Δ·(I + B/2 − 1)]

The Δ in the denominator means: for the same combinatorial polygon (same I, B), a lattice with larger Δ gives a smaller B²/A — it is a more efficient lattice for enclosing area. The square lattice (Δ=1) is more efficient than the Eisenstein (Δ=√3/2) in this sense. But the Eisenstein lattice has more available boundary directions, so its optimal polygon is more circle-like — it can achieve a lower B²/A by choosing a better shape.

The Eisenstein lattice wins the isoperimetric competition. Despite having a smaller fundamental domain (which inflates B²/A for any given combinatorial shape), its 12 available boundary directions allow polygon shapes that more closely approximate a circle. The minimum achievable B²/A on the Eisenstein lattice is actually below 4π for finite polygons — the discrete lattice can beat the continuous bound because the lattice perimeter is not the Euclidean perimeter.

Comparison

Square vs Eisenstein
the isoperimetric comparison

The correct comparison between the two lattices is through B²/A, not B/A. Two effects compete.

Effect 1 — fundamental domain size

For the same combinatorial polygon (same I and B), Pick gives different areas on the two lattices:

Asq = 1·(I + B/2 − 1)    Aei = (√3/2)·(I + B/2 − 1)

Since Aei < Asq, we have B²/Aei > B²/Asq for the same shape. The denser Eisenstein lattice penalises any given combinatorial shape by inflating B²/A.

Effect 2 — available boundary directions

The square lattice has 8 primitive boundary directions (horizontal, vertical, and 4 diagonals). The Eisenstein lattice has 12. More directions → better circle approximation → lower B²/A for the optimal shape. This is the effect that lets the Eisenstein lattice win the isoperimetric competition.

Summary: the Eisenstein lattice has a smaller fundamental domain, which inflates B²/A for any fixed combinatorial polygon. But it compensates by offering more boundary directions, allowing polygon shapes that more closely approximate a circle. The net result is that the optimal Eisenstein polygon achieves B²/A ≈ 12.43, closer to the circle bound 4π ≈ 12.57 than the optimal square lattice polygon at ≈ 13.25.

The Pick thread

The determinant Δ of the lattice basis enters B²/A through the denominator of Pick's formula. It is the same Δ that appeared in the generalised Pick formula A = Δ(I + B/2 − 1). The isoperimetric problem on a lattice is inseparable from the geometry of the basis — which is the central theme of the companion paper.

B²/A = B² / [Δ·(I + B/2 − 1)] — the basis determinant Δ is the lattice ruler
The correct limiting law: for any lattice polygon under dilation, B/A → 0 (trivially). The informative quantity is B²/A, which is constant under dilation and measures how circle-like the polygon is in lattice units. Its minimum over all polygons on a given lattice is the lattice isoperimetric constant, determined jointly by Δ and the available boundary directions.
